配置ununtu防火墙，允许nginx访问端口

允许流量通过nginx

root@Mencius:~# sudo ufw allow 'Nginx Full'
Rules updated
Rules updated (v6)

验证防火墙状态

root@Mencius:~# sudo ufw status
Status: inactive
root@Mencius:~# sudo ufw enable
Command may disrupt existing ssh connections. Proceed with operation (y|n)? y
Firewall is active and enabled on system startup
root@Mencius:~# sudo ufw status
Status: active

To                         Action      From
--                         ------      ----
Nginx Full                 ALLOW       Anywhere                  
Nginx Full (v6)            ALLOW       Anywhere (v6)

验证过程中发现防火墙是关闭的。输入命令启动，然后就可以看到之前配置的防火墙允许nginx通过的配置了。

Chrome浏览器访问"http://116.62.150.136“

访问后提示

无法访问此网站116.62.150.136 的响应时间过长。
请试试以下办法：

检查网络连接
检查代理服务器和防火墙
运行 Windows 网络诊断
ERR_CONNECTION_TIMED_OUT

原因是没有为该服务器配置80端口入站规则，可以在阿里云找到自己的云服务器，为该服务器配置安全组入方向的规则，允许80端口的访问。

接下来再次访问，输入“http://116.62.150.136 ”,可以看到浏览器有如下显示，说明我们可以正常访问我们的nginx了。

Welcome to nginx!
If you see this page, the nginx web server is successfully installed and working. Further configuration is required.

For online documentation and support please refer to nginx.org.
Commercial support is available at nginx.com.

Thank you for using nginx.

关于"https://116.62.150.136"无法访问的问题建议参考一下下面这篇文章

注： 我还没有对其进行验证，验证通过后，我会过程集成到自己的文章中。 https://www.dounaite.com/article/628caae7f8519f4c0cd283e7.html

至此，我在我的阿里云ubuntu服务器上安装nginx,并且访问nginx主页地址就完成了。

但是！！！！我第二次使用Workbench想远程登录的时候却登录不了我的远程服务器。具体描述与解决方案看下面

我第二次登录 阿里云服务器管理控制台，查看某个云服务器ECS实例，点击“远程登录”，选择Workbench远程连接时，却给了一个提示框，提示框中的内容如下：

登录主机: “我的实例名称”失败！
原因：连接超时!
访问公网IP地址需要在实例安全组白名单中增加Workbench的服务器白名单：
47.96.60./24
118.31.243.0/24

于是我去阿里云服务器管理控制台页面的“云服务器/安全组”下配置了入站规则，将上面两个ip允许通过我的服务器,并设置通过端口为22。 但是，还是不能登录！！！！ 经过一番面向四面八方搜索，我发现之前我在安装nginx时将服务器实例的防火墙打开了(详见上面的“验证防火墙状态”章节)，但是没有设置防火墙允许22端口通过外界来得网络数据，为了安全我总不能关闭防火墙吧，后面经过搜索发现，只要执行

sudo ufw allow ssh

就可以让网络流量通过22端口进入到服务器。 但是如果要在服务器上执行上述命令，必须能够登录服务器，而我们现在通过workbench无法登录到我们的远程服务器啊，怎么办，一开始我也是遇到这个问题，其实阿里云服务器管理控制台，查看某个云服务器ECS实例，点击“远程登录”，还有一种以vnc方式登录的方式。以这种方式登录远程服务器，然后执行上面的命令即可。 这里只是通过我们打开阿里的网页，以页面上的workbench的方式远程登录我们服务器，这里限定的ip地址簇为47.96.60./24和118.31.243.0/24。 我想通过本机电脑，通过ssh客户端的方式远程登录我的服务器却登录不了，所以还需要将本机的公网ip地址配置到我的阿里云服务器管理控制台页面的“云服务器/安全组”中去，这样，就可以在本地电脑通过客户端来登录远程服务器了。
