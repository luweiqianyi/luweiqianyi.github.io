markdown放入图片测试

项目构建步骤如下

  • 构建项目平台: Windows
  • 本地的项目根目录为: D:\Hugo\website\
  • 如何构建项目可参考文章：Hugo 搭建自己的博客
  • 在项目根目录的\content\post目录下新建目录blogs_imgs,用来存放博客的图片(目前有一个缺点: 就是所有的博客内的图片都放到该目录下，没有做“博客：博客内引用图片目录=1:1”的区分，后续找到方法后再完善，先这样粗略做一下)
  • 放入图片goInAction.PNG
  • cmd运行hugo构建项目时，会发现D:\Hugo\website\content\post下的文件会被生成到D:\Hugo\website\public\post目录下，所以public目录下就出现了刚才新建的blogs_imgs目录以及该目录下的goInAction.PNG,而生成当前md文档的html页面(运行hugo自动生成)却在post目录下的markdown放入图片测试目录中。所以，进行图片引用的时候需要保证html页面和goInAction.PNG的相对路径正确，即../blogs_imgs/goInAction.PNG
  • 如下所示，如果图片在博客中正常显示(可在本地访问：localhost:1313来看看)，说明上面的描述过程是正确的，否则不正确。
