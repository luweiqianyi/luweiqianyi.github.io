Linux根目录下各目录功能介绍

目录名 目录说明
/ 根目录,以下介绍根目录下的目录及其作用
bin 存放标准的linux工具程序u。当在终端输入ls,系统会去bin目录下查找是否有ls程序
boot Linux内核及引导程序所需要的文件目录
etc 存放系统配置文件方面的文件。当你安装了samba这个套件，当你想要修改samba配置文件的时候，你会发现配置文件就在/etc/samba目录下
dev 存放与设备有关的文件。比如磁盘驱动、USB驱动
home 存放个人数据。用户的设置文件、桌面文件夹、用户数据都在这里。每个用户都有自己的用户目录，位置为/home/用户名。root用户除外。
tmp 临时目录。linux系统会定期清理。
usr 存放不适合放在bin或者etc目录下的额外工具。比如游戏、音乐软件等。/usr/bin:存放程序 /usr/share/：存放共享数据 /usr/lib：存放函数库文件。软件包管理器会自动管理好/usr目录
opt 存放可选程序。比如安装firefox测试版软件到该目录下，当想删掉的时可直接删除，不会影响系统其他设置。
media 挂在usb接口的移动硬盘，CD/DVD驱动器等等。
lost+found 在ext2或ext3文件系统中，当系统意外崩溃或机器意外关机，会产生一些文件碎片放在这里。当系统启动过程中fsck工具会检查这里，并修复已经损坏的文件系统。
mnt 存放“挂在存储设备的挂在目录”。
proc 存放操作系统运行时的进程信息和内核信息。
sbin 存放系统管理命令的存放，是root用户的可执行命令存放地。普通用户无权限执行该目录下的命令。
var 该目录中的内容经常变动。/var/log: 存放系统日志目录 /var/www: 定义Apache服务器站点存放目录 /var/lib:存放一些库文件，比如MySQL的
srv 存放一些服务启动之后需要提取的数据
sys 安装了2.6内核中新出现的文件系统sysfs(集成了针对进程信息的proc文件系统+针对设备的devfs文件系统+针对伪终端的devpts文件系统)
